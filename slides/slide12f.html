Section III-B: Anomaly Detection

Window-Based Real-Time Detection

The same reconstruction idea can be applied continuously on streaming data.

How It Works

The current window is reconstructed at each new timestamp, and its error is checked immediately.

  • Input: a short window W from the sensor stream.
  • Update: the window advances by one timestamp.
  • Output: if RE > Threshold, the current time t0 is marked anomalous.

Why It Matters

Real-time detection raises the time resolution of the analysis, so faults can be caught before the full recording is completed.

Offline detection works at wafer level. Windowed detection pushes the same idea toward in-situ monitoring.
